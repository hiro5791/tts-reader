## 1. このアプリは何か（ゴール）

自分のPCで動く、日本語テキストの読み上げアプリ。

最小版でやりたいことはシンプルに1つだけ：

- テキスト入力欄に文章を貼り付ける
- ボタンを押すと、その文章を音声で読み上げて再生する
- 読み上げエンジンを **Qwen3-TTS** と **Irodori-TTS** から切り替えて、聞き比べられる

それ以外の機能（声の選択、ファイル読み込み、オリジナルの声、誤読の辞書補正など）は、
**この最小版には入れない**。動くものができてから後で足す。

---

## 2. 使うエンジン（重要：ライセンス）

- **Qwen3-TTS** … Apache 2.0（商用利用OK）。0.6B または 1.7B モデル。
- **Irodori-TTS** … 日本語特化。コードは MIT。
  - 注意：Irodori は **日本語のみ対応**。**漢字の読みが弱め**で、難しい漢字は誤読することがある（最小版では許容してよい。辞書補正は後の拡張で対応）。

どちらも、商用でクリーンに使えることが採用理由。最小版では「用意済みの声（プリセット）」で読み上げるだけでよく、
ボイスクローンやVoiceDesignは**まだ実装しない**。

## 3. 動作環境

- 開発機：Windows または Linux の、**NVIDIA GPU 搭載PC**（このPCで動かす前提）
- GPUがある前提で問題ない。CPUのみだと非常に遅いので、GPUを使う構成にする。
- Python を使う（Claude Code が必要なものをインストールしてよい）。

## 4. 作り方の方針（設計）

最小版でも、後の拡張がラクになるよう、最初から「アプリ本体」と「エンジン部分」を分けて作る。

### 4-1. 全体の構成

```
[ UI（テキスト入力＋エンジン選択＋再生ボタン） ]
        ↓ 文章とエンジン名を渡す
[ TTSアダプタ（共通の窓口） ]
        ↓ エンジンごとに振り分け
   ┌──────────────┬──────────────┐
[ Qwen3 アダプタ ]      [ Irodori アダプタ ]
```

- **UI** は、エンジンの細かい違いを知らない。アダプタに「この文章を、このエンジンで」と頼むだけ。
- **TTSアダプタ** が共通の窓口。`synthesize(text, engine) -> 音声ファイル` のような1つの関数を持つ。
- **各エンジンのアダプタ** が、実際に Qwen3 / Irodori を呼び出して音声を作る。
- これにより、後でエンジンを足したり差し替えたりするのが簡単になる。

### 4-2. 共通の窓口（インターフェース）

最小版で必要なのは、実質これ1つだけ：

- `synthesize(text: str, engine: str) -> 音声データ（wavファイルのパス、または音声バイト列）`
  - `engine` は `"qwen3"` か `"irodori"`
  - 中で対応するアダプタを呼び、できた音声を返す

声・速度などのオプションは、最小版では固定でよい（各エンジンのデフォルト＋日本語のプリセット話者）。

### 4-3. UI

- まずは **シンプルなGUI** でよい。実装が簡単な方法を Claude Code が選んでよい
  （例：Python の Gradio で作るローカルWeb画面、または簡単なデスクトップGUI）。
- 画面に必要なもの：
  1. テキスト入力欄（複数行）
  2. エンジン選択（Qwen3 / Irodori を切り替えるラジオボタンかプルダウン）
  3. 「読み上げる」ボタン
  4. 音声プレイヤー（再生バー）
  5. 処理中の表示（「生成中…」など。初回はモデル読み込みで時間がかかるため）

## 5. エンジンの呼び出し方（参考情報）

Claude Code が実装の取っかかりにできるよう、公式の使い方を記す。
最新の正確な使い方は各リポジトリのREADMEを確認すること。

### 5-1. Qwen3-TTS

- パッケージ：`pip install -U qwen-tts soundfile`
- 公式リポジトリ：https://github.com/QwenLM/Qwen3-TTS
- 用意済み話者で読み上げる例（CustomVoiceモデル）。日本語の女性話者 `Ono_Anna` がある：

```python
import torch, soundfile as sf
from qwen_tts import Qwen3TTSModel

model = Qwen3TTSModel.from_pretrained(
    "Qwen/Qwen3-TTS-12Hz-1.7B-CustomVoice",
    device_map="cuda:0",
    dtype=torch.float16,           # GPU(T4等)で安定させるため float16
    attn_implementation="sdpa",    # flash_attention_2 が使えない環境向け
)
wavs, sr = model.generate_custom_voice(
    text="ここに読み上げる日本語",
    language="Japanese",
    speaker="Ono_Anna",
)
sf.write("out_qwen.wav", wavs[0], sr)
```

### 5-2. Irodori-TTS

- 公式リポジトリ：https://github.com/Aratako/Irodori-TTS
- OpenAI互換のAPIサーバー実装もある：https://github.com/Aratako/Irodori-TTS-Server
- コマンド（CLI）で読み上げる例（参照音声なし＝プリセット的に使う）：

```bash
python infer.py \
  --hf-checkpoint Aratako/Irodori-TTS-500M-v3 \
  --text "ここに読み上げる日本語" \
  --no-ref \
  --output-wav out_irodori.wav
```

- Irodori アダプタの実装方法は2通り。Claude Code がやりやすい方を選んでよい：
  1. `infer.py` をコマンドとして呼び出し、出力された wav を読み込む
  2. `Irodori-TTS-Server`（OpenAI互換API）を立てて、HTTPで呼び出す（将来の差し替えがしやすい）

## 7. 最小版では「やらないこと」（スコープ外）

後で足すので、今は実装しない：

- 声の選択・声の種類の切り替え（プリセット1つで固定）
- ボイスクローン、VoiceDesign（オリジナルの声）
- 漢字の誤読を直す読み辞書での前処理
- ファイル（txt / PDF / Word）の読み込み
- 音声ファイルの書き出し（mp3保存）
- クラウドエンジン（Google等）への差し替え
- 速度・ピッチ調整、ハイライト表示 など

※ ただし「アダプタ方式（共通の窓口の裏でエンジンを切り替える構造）」だけは最小版でも採用する。
