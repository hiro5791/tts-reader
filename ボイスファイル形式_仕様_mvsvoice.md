# ボイスファイル形式 仕様（.mvsvoice）

Multi Voice Studio の「ボイスライブラリ」で、1つの声を1ファイルにまとめて保存・持ち運ぶための独自形式の仕様。

> この形式は業界標準ではなく、本アプリ独自に定義するもの。
> 中身は「素材（参照音声＋書き起こし）＋管理情報」なので、Qwen3-TTS でも Irodori-TTS でも使える
> （どちらのエンジンも、参照音声からクローン生成できるため。ただし出力される声はエンジンごとに多少異なる）。

---

## 1. 全体構造

- 拡張子：`.mvsvoice`
- 実体：**ZIPアーカイブ**（中身は下記2ファイル）
- 1つの `.mvsvoice` = 1つの声

```
わたしの声A.mvsvoice   （ZIP）
├── reference.wav      … 参照音声（クローンの素材）
└── voice.json         … 管理情報（名前・書き起こし・権利メモ等）
```

---

## 2. reference.wav（参照音声）

- クローンの素材となる音声。WAV形式。
- 推奨：1人だけが話している／ノイズが少ない／10〜30秒程度／なるべくクリーン。
- これを各エンジンに「参照音声」として渡してクローン生成する。

---

## 3. voice.json（管理情報）

声を再現・管理するための情報。UTF-8 のJSON。

```json
{
  "format_version": 1,
  "name": "わたしの声A",
  "ref_text": "参照音声の中で話している内容をそのまま書いたもの",
  "language": "Japanese",
  "audio_file": "reference.wav",
  "note": "出どころ・許諾の記録（例：自分の声を2026年録音、商用利用可）",
  "created_at": "2026-06-22"
}
```

### 各項目

- `format_version`：形式のバージョン番号（整数）。将来仕様を変えたとき、古い/新しいを見分けるため。現在は `1`。
- `name`：ユーザーが付けた声の名前（表示用）。
- `ref_text`：**参照音声の中で実際に話している内容**を、そのまま文字にしたもの（クローン品質向上のため。Whisper等で自動生成してもよい）。
- `language`：音声の言語（エンジンに渡す正式名。例 `"Japanese"`）。
- `audio_file`：ZIP内の参照音声のファイル名。通常 `"reference.wav"`。
- `note`：出どころ・許諾の記録（商用利用に備えた権利メモ。任意）。
- `created_at`：作成日（任意。`YYYY-MM-DD`）。

---

## 4. アプリでの扱い

### 書き出し（エクスポート）

- ユーザーが「この声を保存」した内容（参照音声＋管理情報）を、
  `reference.wav` と `voice.json` にして、ZIPで固め、`.mvsvoice` として保存する。

### 読み込み（インポート）

- `.mvsvoice`（ZIP）を展開し、`reference.wav` と `voice.json` を取り出す。
- ボイスライブラリに追加する。
- 同名の声が既にある場合は「上書き / 別名で追加」を確認する。
- 不正なファイル・壊れたZIP・必須項目欠落は、エラーを表示して安全に無視（アプリは落とさない）。
  - 必須：`reference.wav` の存在、`voice.json` の `format_version`・`name`・`ref_text`。

### 生成時の使い方

- 声を選んだら、その `reference.wav`（＋ `ref_text`・`language`）を、選択中のエンジンに渡してクローン生成する。
  - Qwen3-TTS：`generate_voice_clone(text=..., language=..., ref_audio=reference.wav, ref_text=ref_text)`
  - Irodori-TTS：`infer.py --ref-wav reference.wav --text ...`（日本語のみ）
- 参照音声は **1回の生成につき1つ**（両エンジンとも、複数同時指定は不可）。

---

## 5. 補足（高速化のキャッシュ・任意）

- 同じ声で何度も生成するなら、エンジンが参照音声を解析した結果（Qwenの再利用プロンプト等）を、
  アプリ内部にキャッシュしてよい（2回目以降の解析を省ける）。
- ただしキャッシュはエンジン固有で、`.mvsvoice` には含めない（`.mvsvoice` はあくまで素材＋管理情報の共通形式）。
- キャッシュが無ければ、`reference.wav` から都度クローンすればよい。

---

## 6. 普段の保存（ライブラリ全体）

- 普段、アプリ内のボイスライブラリは、各声を上記の素材（wav＋json相当）としてローカルに保持し、再起動後も残す。
- `.mvsvoice` は、その1件を「1ファイルにまとめて外部にエクスポート／別PCへ持ち運ぶ」ための形式。
- 複数の声をまとめて渡したい場合は、複数の `.mvsvoice` を渡すか、それらを1つのZIPにまとめてもよい（任意）。
