# Multi Voice Studio 機能仕様書

- **対象アプリ**：Multi Voice Studio（テキスト読み上げ Windows デスクトップアプリ）
- **バージョン**：1.0.1.0（`version.txt` で一元管理）
- **本書の位置づけ**：利用者から見た「何ができ、どう振る舞うか」を機能単位で規定する。
  内部構造・実装方針は [読み上げアプリ_設計書_統合版.md](読み上げアプリ_設計書_統合版.md) を参照。

---

## 1. 概要

テキストを入力すると音声で読み上げるデスクトップアプリ。2つの読み上げエンジン
（**Qwen3-TTS** / **Irodori-TTS**）を切り替えて使える。声・速度・音量・ピッチを調整でき、
プリセット話者のほか「言葉で声を作る（VoiceDesign）」「保存した声（クローン）」に対応する。
生成した音声は再生・波形表示・ファイル保存（wav / mp3）ができ、気に入った声は `.mvsvoice`
形式で保存して再利用できる。UI は20言語に対応する。

- **プラットフォーム**：Windows（CustomTkinter デスクトップアプリ）
- **起動**：`python app.py`（配布版は実行ファイル）
- **オフライン動作**：モデル取得後はネット接続なしで動作可能

---

## 2. 用語

| 用語 | 意味 |
|---|---|
| エンジン | 音声を生成する本体。Qwen3-TTS または Irodori-TTS |
| プリセット（声） | あらかじめ用意された話者・声。プルダウンから選ぶ |
| 声を作る（VoiceDesign） | 声の説明文（例「落ち着いた20代女性の声」）から声を生成する機能 |
| 保存した声（クローン） | 参照音声（`.mvsvoice`）を元に声を再現して読み上げる機能 |
| 喋り方（感情・スタイル） | 読み方の指示（Qwen のみ独立指定可） |
| 音声言語 | 声が「何語として」読み上げるか（表示言語とは別） |
| 表示言語 | アプリ画面の文字の言語（20言語） |
| チャンク | Irodori 長文分割で作られる、約100文字ごとの部分テキスト |

---

## 3. 画面構成

単一のメインウィンドウ。上から下へ以下を配置する。

```
┌──────────────────────────────────────────────┐
│ [エンジン: Qwen3 | Irodori]  音声言語[▼]   表示言語[▼] │  上部バー
├──────────────────────────────────────────────┤
│ [ファイルを開く]  ☑末尾に追加する                    │
│ ┌────────────────────────────────────────┐ │
│ │ テキスト入力欄（複数行・読み上げ中は該当文をハイライト）  │ │
│ └────────────────────────────────────────┘ │
│ ┌ タブ: [プリセット] [声を作る] [保存した声] ─────────┐ │
│ │ (タブごとの入力：声プルダウン／説明文／ファイル選択 等)   │ │
│ └────────────────────────────────────────┘ │
│ 速度[───]  音量[───]  ピッチ[───]                  │  横1行
│ [   読み上げる（生成）   ] [再生] [停止]              │
│ ┌ 波形表示（再生位置カーソル／生成中は進捗バー）───────┐ │
│ └────────────────────────────────────────┘ │
│                      [声を保存する] [音声を保存する] │
│ 状態表示（ステータス／エラー）                        │
└──────────────────────────────────────────────┘
   メニューバー: ヘルプ ＞ 問題を報告 ／ バージョン情報
```

- **速度・音量・ピッチ**は原則横1行。ウィンドウが極端に狭い場合のみ縦並びに組み替える。
- **メニューバー**：ヘルプ ＞「問題を報告」（Googleフォームを開く）／「バージョン情報」（アプリ名・版・注意書き）。

## 5. 機能仕様（詳細）

### F-01 テキスト読み上げ

- **入力**：テキスト入力欄の本文（案内文＝プレースホルダは本文に数えない）。
- **操作**：「読み上げる（生成）」ボタン。
- **挙動**：現在のエンジン・タブ（プリセット/声を作る/保存した声）・声・音声言語・速度・音量・ピッチに従って音声を生成し、生成完了後に自動で再生する。
- **前提**：本文が空のときはエラー表示（生成しない）。
- **非同期**：生成は別スレッドで行い、生成中も画面は固まらない。生成中はテキスト編集・各種操作をロックする（→ F-19 状態遷移）。

### F-02 エンジン切替

- 上部の切替ボタンで **Qwen3-TTS / Irodori-TTS** を選ぶ。
- 切替に連動して、声プルダウン・音声言語プルダウン・喋り方欄の有効/無効が即座に更新される。
- **待機中（idle / ready）のみ切替可**。生成中・再生中は無効。

| | Qwen3-TTS | Irodori-TTS |
|---|---|---|
| 対応言語 | 10言語（日本語含む） | 日本語のみ |
| プリセット話者 | 9種 | デフォルト＋説明文ベース3種 |
| 長文 | 分割せず一括 | 約100文字ごとに自動分割 |
| 喋り方の独立指定 | 可（instruct） | 不可（caption 同居のためグレーアウト） |
| 漢字の読み | 比較的安定 | 弱い（誤読しうる） |

### F-03 声（プリセット）選択

- 「プリセット」タブの声プルダウンから選ぶ。**両エンジンで有効**。
- **Qwen**：`ono_anna`（初期値）/ serena / vivian / sohee / aiden / ryan / dylan / eric / uncle_fu の9種。
- **Irodori**：デフォルト（指定なし）／落ち着いた女性／明るい男性／穏やかで中性的、の4択（うち3種は説明文ベース）。
- 表示名は表示言語に追従する。エンジンを切り替えると選択肢も入れ替わる。

### F-04 音声言語選択

- 上部の音声言語プルダウンで、声が「何語として」読むかを選ぶ。
- **Qwen**：日本語・英語・中国語・韓国語・ドイツ語・フランス語・ロシア語・ポルトガル語・スペイン語・イタリア語の10言語。
- **Irodori**：日本語のみ（選択肢は1件に固定）。
- 音声言語は**表示言語（UIの言語）とは独立**。

### F-05 速度・音量・ピッチ調整

- **速度**：0.5〜2.0倍（初期1.0）。ピッチを保ったまま速さのみ変更。
- **音量**：0〜200%（初期100%）。線形ゲイン後、歪み防止のため [-1,1] にクリップ。
- **ピッチ**：−12〜+12半音（初期0）。長さを保ったまま高さのみ変更。0のときは処理をスキップ。
- 各スライダー右に現在値を表示。**これらは保存対象外**で、毎回既定値から開始する。
- 3値がすべて既定（1.0/100%/0）のときは後処理を行わず、生成音声をそのまま使う。

### F-06 ファイル読み込み

- 「ファイルを開く」ボタンでファイル選択。拡張子で振り分け、本文をテキスト欄に取り込む。

| 種類 | 対応 | 備考 |
|---|---|---|
| `.txt` | ○ | UTF-8 で試し、失敗時は cp932（Shift-JIS）で再試行 |
| `.pdf` | ○ | pypdf で抽出。段組み・表は乱れうる |
| `.docx` | ○ | 段落を改行で連結。旧 `.doc` は非対応 |

- **「末尾に追加する」チェック**：OFF＝置き換え（既定）／ON＝現在の本文の末尾に追記。
- 抽出は完璧ではなく、下書きを流し込む位置づけ（ユーザーが整えてから生成する前提）。

### F-07 声を作る（VoiceDesign）

- 「声を作る」タブの説明文入力欄に、作りたい声を文章で書く（例「落ち着いた20代女性の、温かみのある声」）。
- 「読み上げる」を押すと、説明文から声を作って本文を読み上げる。**Qwen / Irodori 両対応**。
- **Qwen**：VoiceDesign 専用モデルを使用（1.7Bのみ対応）。
- **Irodori**：VoiceDesign 版モデル（caption）を使用。
- **制限**：話者を固定する仕組みが無く、**同じ説明文でも生成ごとに声が微妙にブレる**。気に入った声は F-15 で保存して使い回す。

### F-08 保存した声（クローン）

- 「保存した声」タブで `.mvsvoice` ファイルを **「ファイルを選択」またはドラッグ&ドロップ**で指定する。
- 参照音声を元に声を再現して本文を読み上げる。**両エンジン対応**（Irodori は日本語のみ）。
- **前提**：ファイル未選択のときは生成できない（読み上げボタンが無効）。
- DnD が使えない環境ではファイル選択のみ利用可。

### F-09 喋り方（感情・スタイル）

- 「プリセット」「保存した声」タブに喋り方の入力欄がある。
- **Qwen**：有効。読み方の指示として反映される（プリセットに適用）。
- **Irodori**：**グレーアウト**（感情を独立指定できないため。説明文や本文の絵文字で表現する運用）。
- **既知の制限**：Qwen の「保存した声（クローン）」では、喋り方欄は表示・入力できるが**現状は音声に適用されない**（クローンAPIが指示を受け取らないため）。

### F-10 再生・停止

- **再生**：生成完了後、または「再生」ボタンで最初から再生する。
- **停止**：
  - 再生中 → 再生を止める。
  - 生成中 → 生成を中断する（Irodori は次チャンク境界で停止）。
- 新しい生成・再生を始めると、前の再生は自動的に止まる。

### F-11 読み上げ位置ハイライト

- 再生中、読み上げている**文**の背景を薄い黄色で表示する（文単位）。
- 「。」「！」「？」で文を区切り、各文の文字数比で総再生時間を按分して位置を推定する（近似のため多少ずれる）。
- 停止・一時停止で解除。テキストを編集すると即座に解除する。

### F-12 波形表示・再生カーソル

- 生成後、音声の波形を表示する。
- 再生中は再生位置を示すカーソルが波形上を進む（経過時間ベース）。

### F-13 進捗表示

- **生成中は常に進捗バーを表示**する。
  - Qwen／Irodori 短文（1チャンク）… 左右に動く不確定バー（処理中表示）。
  - Irodori 長文（2チャンク以上に分割）… 「完了数/全数（i/n）」の確定バー＋ステータス表示。
- 正確な件数を出せるのは分割生成のときだけ。

### F-14 音声の保存

- 「音声を保存する」ボタンで、直近に生成した音声を書き出す。
- 保存先を1つ指定すると、**同名で wav と mp3 の両方**を書き出す。
- 生成済み音声が無いときはエラー表示（ボタンも無効）。

### F-15 声の保存（.mvsvoice）

- 「声を保存する」ボタンで、直近に生成した音声を参照音声として `.mvsvoice` に書き出す。
- `.mvsvoice` は zip 形式で、`reference.wav`（参照音声）＋ `voice.json`（メタ情報）を含む。
- 書き出した `.mvsvoice` は F-08「保存した声」で読み込んで再利用できる。
- 形式の詳細は [ボイスファイル形式_仕様_mvsvoice.md](ボイスファイル形式_仕様_mvsvoice.md) を参照。

### F-16 表示言語切替

- 上部の表示言語プルダウンで UI の言語を切り替える。切替時に画面の文字が即座に貼り替わる。
- 対応：日本語 / English / 한국어 / Русский / Українська / हिन्दी / ไทย / 简体中文 / 繁體中文 / Español / Português / Italiano / Deutsch / Nederlands / Polski / Türkçe / Bahasa Indonesia / Tiếng Việt / Français / Svenska（**計20言語**）。
- 初回起動時は OS のロケールから初期言語を自動判定する（対象外なら英語）。

### F-17 設定・ウィンドウ状態の保存

- **保存する設定（`settings.json`）**：エンジン／声（話者）／表示言語／音声言語。
- **ウィンドウ状態（`window_state.json`）**：ウィンドウの位置・サイズ（マルチディスプレイ範囲にクランプ）。
- **保存しない**：速度・音量・ピッチ（毎回既定から開始）、入力テキスト。
- アプリ終了時に保存し、次回起動時に復元する。
- **復元フォールバック**：保存された声がエンジンに無ければ既定の声へ、音声言語が使えなければ既定言語へ、表示言語が対象外なら OS 言語（さらに対象外なら英語）へ。

### F-18 ヘルプ・問題報告

- **バージョン情報**：アプリ名・バージョン・生成物の注意書きを表示するダイアログ。
- **問題を報告**：既定のブラウザで報告用 Google フォームを開く。

## 6. 画面状態遷移（F-19）

画面は4つの状態を持ち、ボタンの有効/無効・進捗バー・ロックがまとめて切り替わる。

| 状態 | 意味 | 読み上げ | 再生 | 停止 | エンジン/声/タブ | テキスト編集 |
|---|---|---|---|---|---|---|
| idle | 起動直後（未生成） | ○※ | × | × | ○ | ○ |
| generating | 生成中 | × | × | ○（中断） | × | ×（ロック） |
| ready | 生成完了・再生待ち | ○※ | ○ | × | ○ | ○ |
| playing | 再生中 | ○※ | × | ○ | × | ○ |

- ※「保存した声」タブでファイル未選択のときは、読み上げは無効。
- 保存ボタン（音声/声）は、生成済み音声があるときのみ有効。
- 生成中はテキスト欄・「ファイルを開く」もロックする（音声と文章のズレ防止）。

遷移の流れ：

```
idle ──[読み上げ]──▶ generating ──[完了]──▶ ready ──[再生]──▶ playing
  ▲                        │                    ▲                │
  └──────[中断/エラー]──────┘                    └────[再生終了]────┘
```

## 7. 入出力ファイル

| 用途 | 形式 | 備考 |
|---|---|---|
| テキスト取り込み（入力） | `.txt` / `.pdf` / `.docx` | F-06 |
| 生成音声（出力） | `.wav` ＋ `.mp3` | F-14。両方同時に書き出し |
| 声（入出力） | `.mvsvoice`（zip） | F-08 / F-15 |
| 設定 | `settings.json` | F-17 |
| ウィンドウ状態 | `window_state.json` | F-17 |
| クラッシュ記録 | `logs/crash.log` | 障害解析用（自動生成） |

- 出力・設定・ログの保存先は書き込み可能なデータフォルダ。配布版は `%LOCALAPPDATA%\MultiVoiceStudio\`、開発時はプロジェクト直下。

## 8. エラー・例外時の挙動

- **本文が空**：エラー表示し、生成しない。
- **保存した声：ファイル未選択**：エラー表示（読み上げボタンも無効）。
- **保存対象の音声が無い状態で保存**：エラー表示（ボタンも無効）。
- **生成中の中断**：停止ボタンで中断し、結果は破棄する。
- **ファイル読み込み失敗（文字コード等）**：txt は UTF-8→cp932 の順で自動再試行。
- **モデル/生成の失敗**：エラー内容をステータスに表示する。
- **VRAM 不足・GPU 非搭載**：自動的に CPU 実行にフォールバックする（遅いが停止せず動作）。クローン（float32・重い）は特に CPU になりやすい。
- **致命的クラッシュ**：`logs/crash.log` にスタックを記録する（無言終了の解析用）。

## 9. 動作環境

- **OS**：Windows。
- **GPU**：NVIDIA GPU があれば高速。無い／VRAM 不足でも CPU で動作（遅い）。
- **メモリ/VRAM**：モードごとに別モデルを読むため、使わないモデルは自動解放して同時ロードを実質1つに抑える（8GB 級 GPU 対策）。
- **ネットワーク**：初回のモデル取得時のみ必要。取得後はオフライン動作可。

---

## 10. 非機能・制約・既知の制限

- **性能**：初回のモデル読み込みに時間がかかる（都度ではなくエンジン/モード切替時）。生成は別スレッドで実行し UI は固まらない。
- **多言語**：UI は20言語、音声言語は Qwen が10言語・Irodori が日本語のみ。
- **Irodori の制限**：日本語のみ／漢字の読みが弱い（誤読しうる）／喋り方の独立指定不可／長文は自動分割（つなぎ目に約0.25秒の無音）。
- **VoiceDesign の制限**：生成ごとに声が微妙にブレる（話者固定機構なし）。→ 保存して再利用。
- **クローン＋喋り方**：Qwen のクローンでは喋り方欄は適用されない（現状の制限）。
- **ハイライト**：文単位の近似（文字数按分）のため多少のずれが生じる。
- **PDF/Word 抽出**：レイアウト・表は乱れうる。下書き用途。
