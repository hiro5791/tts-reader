# =============================================================================
# 配布ビルド用の「統合環境」依存リスト
#   目的: app.py / Qwen3-TTS / Irodori-TTS を 1 つの Python 環境に入れて、
#        Irodori を uv 別プロセスでなく「同一プロセス（in-process）」で動かす。
#        これにより PyInstaller で 1 つにまとめてストア配布できる。
#
#   ※ 開発用の .venv（requirements.txt）とは別に、専用のビルド環境を作って使うこと。
#      作業中のアプリ環境を壊さないため。
#
# -----------------------------------------------------------------------------
# 手順:
#   1) 新しい仮想環境を作る（例）:  py -3.11 -m venv .venv-build
#   2) torch 系を先に CUDA 版で入れる（Irodori の torchcodec<0.11 に合わせ 2.10 系で統一）:
#        pip install "torch==2.10.*" "torchaudio==2.10.*" "torchcodec==0.10.*" \
#            --index-url https://download.pytorch.org/whl/cu128
#      （qwen-tts / transformers は torch のバージョンに寛容なので 2.10 で動く見込み。
#        実機で Qwen / Irodori 両方の生成が通ることを必ず確認すること）
#   3) 残りを入れる:  pip install -r requirements-build.txt
#   4) モデルを取得:  python scripts/prefetch_models.py
#   5) 動作確認:  python app.py で Qwen と Irodori 両方の生成を確認
#        （Irodori が "in-process" ログで動けば脱 uv 成功。uv は不要）
# =============================================================================
